# DevDromos - Συνολική Τεκμηρίωση Μέρους Β (Client-Server)

Αυτό το αρχείο περιγράφει τη συνολική αρχιτεκτονική και λειτουργία της εφαρμογής DevDromos για το Μέρος Β, καλύπτοντας την επικοινωνία μεταξύ του Frontend (Client) και του Backend (Server).

## Αρχιτεκτονική & Ροή Δεδομένων
Η εφαρμογή ακολουθεί το μοντέλο **REST API**. 
1. **Frontend (Vite + TypeScript):** Ο χρήστης αλληλεπιδρά με τη διεπαφή. Το TypeScript στέλνει ασύγχρονα αιτήματα (fetch) στον server.
2. **Backend (Node.js + Express):** Ο server λαμβάνει τα αιτήματα, ελέγχει την εγκυρότητα του Firebase ID token (αν απαιτείται) και επικοινωνεί με τη βάση.
3. **Βάση Δεδομένων (Firebase/Firestore):** Αποθηκεύει μόνιμα τους χρήστες, τα μαθήματα, τα βιβλία και τα καλάθια αγορών.

## Κύρια Endpoints (API)
Η επικοινωνία βασίζεται στα παρακάτω κεντρικά σημεία:

| Κατηγορία | Endpoint | Λειτουργία |
|-----------|----------|------------|
| **Auth** | `POST /api/auth/register` | Εγγραφή νέου χρήστη & αυτόματο login με Firebase |
| **Auth** | `POST /api/auth/login` | Ταυτοποίηση & σύνδεση χρήστη |
| **Data** | `GET /api/courses` | Ανάκτηση όλων των μαθημάτων |
| **Data** | `GET /api/books` | Ανάκτηση όλων των βιβλίων |
| **Cart** | `GET /api/cart` | Προβολή αντικειμένων στο καλάθι (απαιτεί Firebase ID token) |
| **Cart** | `POST /api/cart` | Προσθήκη αντικειμένου στο καλάθι |
| **Orders**| `POST /api/checkout` | Ολοκλήρωση αγοράς & δημιουργία εγγραφών |

## Ελάχιστη Λειτουργικότητα (Minimum Viable Product)
Βάσει των απαιτήσεων, υλοποιήθηκαν τα εξής:
* **Πλήρες Authentication:** Εγγραφή και Σύνδεση με Firebase Auth και διατήρηση session μέσω Firebase ID token στο LocalStorage.
* **Δυναμική Διαχείριση Περιεχομένου:** Τα δεδομένα δεν είναι στατικά, αλλά έρχονται από το Firestore.
* **Σύστημα Καλαθιού:** Ο χρήστης μπορεί να προσθέτει/αφαιρεί προϊόντα, τα οποία διατηρούνται στη βάση δεδομένων ακόμα και μετά από refresh της σελίδας.
* **Προστασία Δεδομένων:** Σελίδες όπως το "Cart" και η διαδικασία "Checkout" είναι προσβάσιμες μόνο σε συνδεδεμένους χρήστες (Auth Guards).

## Οδηγίες Εκτέλεσης
1. Ρυθμίστε το Firebase project και το service account `.env` στον φάκελο `server`.
2. Στον φάκελο `server`, εκτελέστε `npm install` και `npm start` (στη θύρα 4000).
3. Στον φάκελο `client`, εκτελέστε `npm install` και `npm run dev` (στη θύρα 5173).
